# 全局执行原则

## 可恢复

所有项目都必须通过自身的 STATE、TASKS、CORE、LOGS 恢复上下文，不依赖 AI 的临时对话记忆。

## 可交接

任何 AI 接入项目时，都必须能通过固定文件顺序理解当前项目、当前状态、当前任务和边界。

## 可隔离

每个项目都是独立实例。项目之间不得共享状态、任务、日志或业务认知。

## 可追踪

所有已发生的重要操作必须写入项目自己的 LOGS 文件。全局层不记录项目执行日志。

## 最小修改

AI 只能修改完成当前授权任务所需的最小文件集合，不改动无关项目和无关业务代码。

## 权限优先

当用户授权范围、项目规则和执行任务存在冲突时，必须先停止并澄清，不得擅自扩大操作范围。
