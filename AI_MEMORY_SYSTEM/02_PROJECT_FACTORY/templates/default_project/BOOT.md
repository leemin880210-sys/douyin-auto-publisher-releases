# 项目 BOOT

本文件是项目实例入口。任何 AI 进入本项目时，必须先读取本文件，再按固定顺序执行。

## 执行顺序

必须严格按以下顺序读取和执行：

1. `BOOT.md`：确认项目入口、边界和规则。
2. `STATE.json`：确认当前唯一状态。
3. `TASKS.json`：确认下一步任务。
4. `CORE.md`：确认项目长期认知和业务边界。
5. `LOGS.md`：确认历史执行事实。
6. 执行用户授权任务。
7. 回写 `STATE.json` 和 `LOGS.md`，并在任务变化时同步 `TASKS.json`。

简写顺序：BOOT → STATE → TASKS → CORE → LOGS。

## 隔离规则

- 不允许跨项目读取数据。
- 不允许共享 STATE。
- 不允许修改未授权项目。
- 本项目的状态、任务、认知和日志只属于本项目。
