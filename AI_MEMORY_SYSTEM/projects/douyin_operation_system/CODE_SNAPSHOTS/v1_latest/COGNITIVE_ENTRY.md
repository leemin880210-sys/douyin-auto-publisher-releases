# 🧠 COGNITIVE ENTRY（认知统一入口）

本文件是整个外部大脑系统的唯一“认知入口”。

👉 新AI必须先读取本文件，才能理解整个系统。

---

# 1. 系统是什么

这是一个：

👉 抖音代运营 AI 操作系统

不是采集工具，不是脚本，不是单模块系统。

---

# 2. 当前系统真实状态

当前系统处于：

👉 account_ops 账号采集阶段

说明：

- 只做账号采集
- 不做账号分析
- 不做商家建档
- 不做内容生成
- 不做复盘

---

# 3. 系统结构（统一理解）

系统由四层组成：

## 🧠 第一层：控制层

MASTER_CONTROL
→ 决定“能不能做”

---

## 🧠 第二层：状态层

STATE.json
→ 当前正在做什么

---

## 🧠 第三层：任务层

TASKS.json
→ 下一步要做什么

---

## 🧠 第四层：结构层

PROJECT_FRAMEWORK + MODULE_ROUTES
→ 系统整体设计

---

# 4. 关键原则（必须遵守）

## ❗原则1：STATE只描述现在

不能包含未来规划

---

## ❗原则2：FRAMEWORK只描述结构

不能描述当前状态

---

## ❗原则3：TASKS只描述下一步

不能描述系统结构

---

## ❗原则4：MASTER_CONTROL只控制权限

不能描述业务内容

---

# 5. AI进入系统必须做什么

新AI必须：

1. 先读本文件
2. 再读 STATE.json
3. 再读 TASKS.json
4. 再读 MASTER_CONTROL
5. 再读 FRAMEWORK

---

# 6. 系统的本质

这是一个：

👉 “可跨AI接管的抖音代运营操作系统”

核心目标：

- 不从0开始
- 不重复设计
- 不丢失历史
- 不重构逻辑

---

# 7. 当前禁止行为

任何AI禁止：

- 跨模块执行
- 跳过采集直接分析
- 自动创建商家系统
- 自动生成运营方案
- 修改系统结构

---

# 8. 为什么需要这个文件

因为之前系统存在：

- 多文件认知冲突
- STATE语义扩展
- FRAMEWORK误导执行
- AI从0理解问题

本文件用于：

👉 统一所有AI的“第一认知入口”

---

# 9. 最重要规则（终极）

如果一个AI只读这一份文件：

👉 也必须能理解整个系统在做什么

---

# 10. 文件约束声明

- 其他外部大脑说明文件受 `COGNITIVE_ENTRY.md` 约束。
- `STATE.json` 是运行状态唯一来源，受 `COGNITIVE_ENTRY.md` 约束。
- `TASKS.json` 是任务唯一来源，受 `COGNITIVE_ENTRY.md` 约束。
- `MASTER_CONTROL.md` 是执行权限来源，受 `COGNITIVE_ENTRY.md` 约束。
- `PROJECT_FRAMEWORK.md` 与 `MODULE_ROUTES.md` 只解释结构和路由，受 `COGNITIVE_ENTRY.md` 约束。
