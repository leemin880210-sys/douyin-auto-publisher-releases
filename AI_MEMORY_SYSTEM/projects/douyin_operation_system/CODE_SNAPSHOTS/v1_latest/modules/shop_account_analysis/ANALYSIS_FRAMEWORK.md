# ANALYSIS_FRAMEWORK

## 分析目标

根据 `account_ops` 生成的采集包，对店铺抖音账号进行接手前深度诊断。

## 分析维度

1. 账号基础判断：昵称、简介、主页入口、作品数量、公开互动线索。
2. 主页问题：定位是否清楚、转化入口是否明确、主页信息是否完整。
3. 作品问题：选题、画面、标题、节奏、垂直度、转化证据。
4. 评论区用户需求：咨询、价格、地址、体验、疑虑、复购线索。
5. 内容垂直度：账号是否围绕稳定业务主题输出。
6. 本地转化问题：地址、团购、预约、私信、营业时间等证据是否清楚。
7. 接手方向：先修主页、先做内容、先补素材、先补商家资料等。
8. 接手风险：数据不足、评论缺失、OCR 不稳定、授权指标缺失等。
9. 商家补充资料：地址、菜单、价格、活动、素材、目标客群、服务范围。

## 输出原则

- 只基于采集包证据分析。
- 不编造授权指标。
- 不生成视频脚本。
- 不创建商家长期大脑。
- 不把采集包本体写入 AI_MEMORY_SYSTEM。
