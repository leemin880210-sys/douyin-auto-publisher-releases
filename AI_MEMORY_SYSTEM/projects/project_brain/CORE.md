# 项目认知

本文件是当前项目的长期核心认知。除非项目方向重构，否则不要频繁修改。

## 在全局架构中的位置

本项目是 `AI_MEMORY_SYSTEM/projects/project_brain` 下的单项目实例。

本项目只保存自身的业务认知、状态规则、任务规则和项目边界，不保存全局 AI 规则，不承担项目注册中心职责。

## 本项目做什么

本项目是一个面向微小商家的抖音代运营、数据采集与 AI 工作流系统。当前重点包括：

- 抖音账号公开内容采集与分析包生成
- 商家建档
- 素材整理
- 短视频脚本生成
- 视频初稿与人工审核
- 发布支持
- 数据复盘
- GPT 与 Codex 协同执行

## 服务对象是谁

服务对象是本地生活、餐饮、花艺、门店服务等类型的微小商家。项目面向的是预算有限、需要内容运营支持、但不一定具备完整运营团队的小商家。

## 核心目标是什么

核心目标是建立一个可恢复、可交接、可持续执行的 AI 代运营工作流：

1. 用本地工具整理账号公开内容资料包。
2. 用 GPT 分析商家账号、内容、评论和转化线索。
3. 用 Codex 持续维护工具、流程和交付结构。
4. 用本项目记忆文件让任何授权 AI 接入后能快速恢复项目状态。

## 抖音项目核心能力

本项目用于：

- 抖音代运营内容生成
- 商家建档
- 视频脚本生成
- 视频、图片、zip 输出
- 数据复盘
- AI + Codex 协同执行

## 输出命名规则（v1.0）

规则版本时间：`2026-01-24_1530`

所有输出必须统一命名：

```text
{店铺名称}-{作品数量}-{时间}
```

时间格式：

```text
YYYYMMDD_HHMM
```

示例：

```text
星火奶茶店-001-20260124_1530.mp4
星火奶茶店-002-20260124_1530.txt
```

## ZIP 压缩包规则

所有 zip 文件必须存放在：

```text
/output_zip/
```

zip 命名规则：

```text
{店铺名称}-{作品数量}-{时间}.zip
```

示例：

```text
/output_zip/星火奶茶店-003-20260124_1530.zip
```

## 防冲突规则

- 不允许同名覆盖。
- 每个作品必须递增编号。
- 同商家必须连续编号。
- 不同商家必须隔离命名。
- 所有输出必须唯一。

## 当前运行模式

当前系统运行模式为：

```text
人工触发 + AI生成 + 结构化输出模式
```

当前不是自动执行系统。

## 当前已具备能力

- 多商家并行处理
- 自动编号系统
- ZIP 独立目录输出
- 防重名机制
- 标准化文件体系
- 可追溯记录结构

## 当前限制

- 未实现自动文件夹隔离（每商家独立目录）。
- 未实现自动上传/同步机制。

## 禁止做什么

- 不绕过登录。
- 不破解验证码。
- 不抓取无权限内容。
- 不采集或外传本机 cookie。
- 不把未授权数据伪造成已授权数据。
- 不承诺投流、广告投放、涨粉或成交结果。
- 不修改与当前任务无关的业务代码。
- 不删除历史日志。
- 不跨项目读取或复用其他项目 brain。
- 不覆盖已有输出文件。
- 不混用不同商家的作品编号。

## Codex 工作规则

Codex 在本项目中不是普通开发者，而是严格按状态机执行的操作单元。

Codex 接入项目后必须按以下顺序执行：

1. 读取 `BOOT.md`。
2. 读取 `STATE.json`。
3. 读取 `TASKS.json`。
4. 读取 `CORE.md`。
5. 明确当前任务是否允许修改业务代码。
6. 执行任务时只修改允许范围内的文件。
7. 任务结束前更新 `STATE.json`。
8. 任务结束前写入 `LOGS.md`。
9. 任务变化时同步 `TASKS.json`。
10. 任务结束后不强制生成本地交付包；需要外部检查时优先同步 GitHub 上的 `AI_MEMORY_SYSTEM/projects/project_brain/`。

如果 STATE 更新、LOG 记录、必要的 TASK 同步没有完成，Codex 不允许返回完成状态。

## GPT 检查规则

GPT 接入项目后必须：

1. 先读取 `BOOT.md`。
2. 判断当前项目在做什么。
3. 判断当前状态是否与用户任务一致。
4. 检查 Codex 输出是否符合验收标准。
5. 不允许凭空推断未写入 STATE 或 TASKS 的状态。
6. 不允许忽略任务优先级。
7. 不允许跨项目使用记忆。
8. 检查输出文件是否符合统一命名、连续编号、ZIP 目录和防冲突规则。
