# Code Evolution Memory（代码演进日志）

## 作用
记录本项目所有关键代码变更，保证 Codex 在每次修改时可以追踪历史演进，避免重复修改或结构断裂。

---

## 规则

1. 每次修改代码必须追加一条记录。
2. 不允许覆盖历史记录。
3. 必须包含：
   - 修改时间
   - 修改内容
   - 修改原因
   - 修改文件路径
   - 修改前状态
   - 修改后状态

---

## 示例记录格式

### 2026-06-27

- 修改内容：优化脚本生成逻辑
- 修改原因：提升稳定性
- 修改路径：/script/generator.py
- 修改前：使用旧模板A
- 修改后：使用模板B
- 备注：禁止回滚旧逻辑

---

## 强制约束

- Codex 在修改任何代码前必须读取本文件
- 修改后必须追加记录
- 不允许跳过记录步骤
